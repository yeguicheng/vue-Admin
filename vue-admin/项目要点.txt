一、用户登录页面：
    1、使用了element-ui插件
    2、登录与注册两个数据通过定义数据(数组对象)来渲染,其中数据中存放着当前显示的状态 “CurrentFlag(布尔值)”,通过点击方法(方法中传递当前对象‘ 引用类型，会相互影响 ’数据作为参数)来改变状态
    3、单独将表单验证的方法规则封装在一个文件夹中(其中方法返回布尔值)
二、注册页面
    1、由于登录与注册的表单差不多相同，所以会共用表单。但是会多出一个‘重复密码表单框’,这时可以通过用'v-show'来判定数据(可以在数据中设置两个状态)，这里有一个问题，因为使用‘v-show’表单始终会存在dom对象中，这时验证规则也会验证‘它’。
    因此，需要在验证规则中进行判定状态，即状态为login时，不用验证，直接通过
三、将vue2.0修改为3.0
    1、将2.0中的声明方式改用了setup(props,context)函数式
        1.1、setup(){
            可以存放2.0中data的数据、自定义函数、钩子函数、
        }
    2、在data中引用数据类型改用：
        2、1 const xxx = reactive(数据类型)
    3、在data中基本数据类型改用：
        3、1 const xxx = ref(基本数据)
    4、可以使用“isref()”来判断是否为基本数据类型
    5、可以使用“toRefs()”来将reactive类型转换为普通对象类型
    6、钩子函数要变为“onMounted”
